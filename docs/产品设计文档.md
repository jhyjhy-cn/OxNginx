# Nginx Manager（Rust Edition）产品设计文档

> 一个专注于 **轻量、高性能、低内存占用** 的 Nginx 可视化管理面板。

---

# 一、项目定位

## 项目目标

开发一个专门用于管理 Nginx 的可视化面板，主要提供：

* Nginx 站点管理
* 反向代理管理
* SSL 证书管理
* HTTP/HTTPS 配置
* TCP(Stream)代理
* 配置文件编辑
* 一键检测配置
* 一键 Reload
* 配置历史版本
* 自动回滚

目标用户：

* 个人开发者
* 小型 VPS（1C1G / 2C2G）
* x-ui 用户
* Reality/VLESS 用户
* HomeLab 用户

---

# 二、核心设计理念

## 极致轻量

目标：

* 常驻内存 < 20MB
* CPU 几乎为 0%
* 单个可执行文件
* 无 Docker 依赖
* 无 Node Runtime
* 无 JVM
* 无 Python Runtime

部署后：

```
Browser
    │
    ▼
Rust Binary
    │
SQLite
    │
Nginx
```

整个程序只有：

```
一个 Binary
一个 SQLite
一个 systemd Service
```

---

## 不做重后台

不采用 Nginx UI 的设计。

禁止：

* 文件监听(File Watcher)
* 配置索引
* 日志索引
* Analytics
* EventBus
* WebSocket Hub
* 全局缓存
* 全文搜索

所有数据均按需读取。

例如：

```
点击站点列表

↓

读取 nginx 配置

↓

返回 JSON

↓

结束
```

而不是后台一直扫描。

# 十五、Claude Code 开发要求

## 开发原则

1. 采用 Clean Architecture（API → Service → Repository → Infrastructure）。
2. 保持模块高内聚、低耦合，避免所有逻辑集中在 `main.rs`。
3. API 返回统一 JSON 结构（包含 `code`、`message`、`data`）。
4. 所有配置生成采用模板方式，禁止字符串拼接。
5. 所有配置变更必须经过：

   * 生成配置
   * 备份旧配置
   * `nginx -t`
   * 成功后 Reload
   * 失败自动回滚
6. 所有耗时任务（证书申请、备份、日志读取等）采用 Tokio 异步执行。
7. 前端与后端分离开发，最终将 Vue 打包产物嵌入 Rust 二进制发布。
8. 项目需提供完整的单元测试、集成测试及 GitHub Actions CI。
9. 代码遵循 Rust 官方最佳实践，避免 `unwrap()`，统一错误处理。
10. 目标是打造一个**比 Nginx UI 更轻、更快、更稳定、更适合小型 VPS 的现代化 Nginx 管理面板**。
